html_data

JSON representation of HTML

Other properties besides tag and inner are used to define HTML attributes. A string instead of an object represents pure text content without HTML tags. The HTML data can also contain placeholders marked with %%, which can be dynamically replaced with values via ccm.helper.html or ccm.helper.format.

Type:
  • Object | string
Properties:
Name Type Argument Default Description

HTML tag name
inner ccm.types.html_data <optional>

{
  tag: "p",
  inner: [
    "Hello, ",
    {
      tag: "b",
      inner: "%name%",

    },

resource_obj

Instead of a URL, a resource object can be passed to the method ccm.load, which then contains other information besides the URL, via which the loading of the resource is even more flexible controllable. In the case of HTML, JSON and XML, the Fetch API is used to load the ressource. All properties in the resource object are also spread into the options object (2nd parameter of fetch()). This means that, for example, HTTP headers can also be set here.

Type:
  • Object
Properties:
Name Type Argument Description
url string

URL from which the resource should be loaded.
context Element | ccm.types.instance <optional>

Context in which the resource is loaded (default is <head>). Only relevant when loading CSS or JavaScript. CSS is loaded via <link> and JavaScript is loaded via <script>. When a ccmjs component instance is passed, the resource is loaded in the Shadow DOM of that instance.
type string <optional>

Resource is loaded as 'css', 'html', 'image', 'js', 'module', 'json' or 'xml'. If not specified, the type is automatically recognized by the file extension. If the file extension is unknown, 'json' is used by default.
attr string <optional>

Additional HTML attributes to be set for the HTML tag that loads the resource. Only relevant when loading CSS or JavaScript. CSS is loaded via <link> and JavaScript is loaded via <script>. With the additional attributes integrity and crossorigin the resource can be loaded with Subresource Integrity (SRI).
method string <optional>

The request method, e.g., "GET", "POST". The default is "GET". Only relevant when loading data. "JSONP" is also supported.
params string <optional>

HTTP parameters to send. Only relevant when loading data.
